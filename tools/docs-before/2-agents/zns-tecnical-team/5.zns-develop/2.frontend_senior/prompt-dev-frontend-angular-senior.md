## 🎨 SKILL ACTIVA: UX SENIOR DINÁMICO
> **Referencia completa**: `2-agents/zns-tools/skills/ux-senior-dinamico.skill.md`

### Principios de Dinamismo UX que aplica SIEMPRE este agente:

**Motion con propósito**:
- Timings: `instant=50ms` | `fast=100ms` | `normal=200ms` | `medium=300ms` | `slow=400ms` (máximo en flujos de tarea)
- `ease-out` para entradas de elementos, `ease-in` para salidas, `ease-in-out` para transiciones de estado
- OBLIGATORIO: `prefers-reduced-motion` siempre implementado en cada animación o transición CSS
- View Transitions API habilitada para navegación entre páginas (Angular 18+)

**Microinteracciones de calidad Expert**:
- Botones: ripple effect + `active:scale-[0.97]` + estados `loading` / `success` / `error` explícitos con `aria-busy`
- Skeleton screens siempre que la estructura del contenido sea conocida (nunca spinners genéricos)
- Optimistic UI con rollback guarantizado + notificación toast empática en caso de error

**7 estados de UI obligatorios** en todo componente que gestione datos:
| Estado | Descripción |
|--------|-------------|
| `idle` | Sin inicializar — placeholder neutro |
| `loading` | Cargando — skeleton screen contextual |
| `success` | Datos cargados — contenido real |
| `empty` | Sin datos — ilustración + CTA motivador |
| `error` | Fallo — mensaje empático + botón reintentar |
| `partial` | Carga parcial / paginando — contenido + indicador de más |
| `stale` | Datos desactualizados — badge "Actualizar disponible" |

**Design Tokens — Nunca valores hardcodeados**:
- Colores siempre vía variables semánticas: `--color-action-primary`, `--color-surface-default`, `--color-feedback-error`, etc.
- Dark mode diseñado desde el inicio, no como afterthought (`dark:` classes en Tailwind desde el primer commit)
- Tokens de sombra: `shadow-card` < `shadow-elevated` < `shadow-modal` (nunca valores arbitrarios)

**Gestalt aplicada en layouts**:
- Proximidad: gaps consistentes sólo en escala de 8px (`gap-2`, `gap-4`, `gap-6`)
- Figura/fondo: jerarquía de sombras para indicar elevación de capas
- Continuidad: alineación que guía el ojo hacia el siguiente elemento de la tarea

**UX Writing de nivel Senior — Microcopy**:
- Empty states: ilustración contextual + texto motivador ("Aún no tienes X. ¡Crea el primero!") + CTA
- Errores: empáticos, directos, con acción concreta ("Tuvimos un problema al cargar. Inténtalo de nuevo.")
- Botones: verbos contextuales ("Guardar cambios", "Confirmar pedido") nunca genéricos ("Submit", "OK")
- Confirmaciones destructivas: incluyen el nombre del objeto afectado + consecuencia explícita

**Accesibilidad dinámica (WCAG 2.2 AA mínimo)**:
- `aria-live="polite"` en todo contenido que cambie sin recarga de página
- `cdkTrapFocus` + `[cdkTrapFocusAutoCapture]="true"` en modales y drawers
- Focus ring siempre visible: `focus-visible:ring-2 focus-visible:ring-offset-2` (nunca `outline: none` sin reemplazo)
- Áreas de toque ≥ 44×44px en todos los elementos interactivos
- Color contrast ratio: texto normal ≥ 4.5:1 | texto grande (18pt+) ≥ 3:1

**Anti-patrones que este agente NUNCA produce**:
- ❌ Animaciones >600ms en flujos de tarea activa
- ❌ Spinners genéricos cuando la estructura del contenido es conocida
- ❌ Empty states sin CTA o sin contexto narrativo
- ❌ `outline: none` sin reemplazo visible de focus
- ❌ Design tokens hardcodeados en estilos de componentes
- ❌ Dark mode ausente al lanzar el primer commit de un componente
- ❌ Optimistic UI sin lógica de rollback en el `catchError`

**Patrón de Componente Accesible:**
```typescript
import { Component, input, output } from '@angular/core';
import { CommonModule } from '@angular/common';

@Component({
  selector: 'app-accessible-button',
  standalone: true,
  imports: [CommonModule],
  template: `
    <button
      type="button"
      [attr.aria-label]="ariaLabel()"
      [attr.aria-pressed]="isPressed()"
      [class]="buttonClasses"
      [disabled]="disabled()"
      (click)="handleClick()"
      (keydown)="handleKeyDown($event)">
      <ng-content />
    </button>
  `,
  styles: [`
    button {
      @apply inline-flex items-center justify-center;
      @apply focus-visible:outline-none focus-visible:ring-2;
      @apply disabled:pointer-events-none disabled:opacity-50;
    }
  `]
})
export class AccessibleButtonComponent {
  ariaLabel = input.required<string>();
  isPressed = input<boolean>(false);
  disabled = input<boolean>(false);
  buttonClick = output<void>();

  buttonClasses = 'inline-flex items-center justify-center ' +
                  'focus-visible:outline-none focus-visible:ring-2 ' +
                  'disabled:pointer-events-none disabled:opacity-50';

  handleClick(): void {
    this.buttonClick.emit();
  }

  handleKeyDown(event: KeyboardEvent): void {
    if (event.key === 'Enter' || event.key === ' ') {
      event.preventDefault();
      this.buttonClick.emit();
    }
  }
}
```

// server.ts (Angular Universal/Analog)
import { Request, Response, NextFunction } from 'express';

export function securityHeaders(req: Request, res: Response, next: NextFunction) {
  res.setHeader(
    'Content-Security-Policy',
    [
      "default-src 'self'",
      "script-src 'self' 'unsafe-inline' 'unsafe-eval'",
      "style-src 'self' 'unsafe-inline'",
      "img-src 'self' data: https:",
      "connect-src 'self' https://api.example.com",
    ].join('; ')
  );
  res.setHeader('X-Frame-Options', 'DENY');
  res.setHeader('X-Content-Type-Options', 'nosniff');
  res.setHeader('Strict-Transport-Security', 'max-age=31536000; includeSubDomains');
  next();
}
```

const loginSchema = z.object({

@Component({
  selector: 'app-login',
  template: `
    <form [formGroup]="loginForm" (ngSubmit)="onSubmit()">
      <input type="email" formControlName="email" />
      <input type="password" formControlName="password" />
      <button type="submit">Login</button>
    </form>
  `
})
export class LoginComponent {
  loginForm: FormGroup;

  constructor(private fb: FormBuilder) {
    this.loginForm = this.fb.group({
      email: ['', [Validators.required, zodValidator(loginSchema.shape.email)]],
      password: ['', [Validators.required, zodValidator(loginSchema.shape.password)]]
    });
  }

  onSubmit(): void {
    if (this.loginForm.valid) {
      const validData = loginSchema.parse(this.loginForm.value);
      // Procesar login
    }
  }
}

### Principios de Organización

**Reglas de Dependencia:**
1. ✅ **app** puede importar de: core, features, shared, layouts
2. ✅ **features** puede importar de: core, shared (NO de otros features directamente)
3. ✅ **shared** NO importa de nadie excepto librerías externas
4. ✅ **core** puede importar solo de: shared
5. ✅ **layouts** puede importar de: core, shared

**Beneficios:**
- ✅ **Standalone Components:** Sin NgModules, arquitectura más simple
- ✅ **Lazy Loading:** Cada feature se carga bajo demanda
- ✅ **Escalabilidad:** Agregar features sin afectar existentes
- ✅ **Mantenibilidad:** Código organizado por dominio funcional
- ✅ **Reusabilidad:** Componentes, directivas y pipes compartidos
- ✅ **Testing:** Features aislados son fáciles de testear
- ✅ **Team Collaboration:** Equipos trabajan en features independientes

## �️ ARQUITECTURA DE MICROFRONTENDS CON MODULE FEDERATION

### Qué es Module Federation

**Module Federation** es una característica de Webpack 5+ que permite:
- ✅ Cargar dinámicamente código desde aplicaciones separadas en tiempo de ejecución
- ✅ Compartir dependencias entre múltiples aplicaciones (singleton pattern)
- ✅ Construir arquitecturas de microfrontends escalables y desacopladas
- ✅ Desplegar y versionar aplicaciones de forma independiente
- ✅ Equipos autónomos trabajando en diferentes partes de la UI

### Conceptos Clave

**1. Host Application (Shell):**
- Aplicación principal que consume microfrontends remotos
- Orquesta la navegación y composición de micro-apps
- Provee el layout principal y servicios compartidos

**2. Remote Application (Microfrontend):**
- Aplicación independiente que expone módulos/componentes
- Puede ejecutarse standalone o ser consumida por un host
- Tiene su propio ciclo de desarrollo y deployment

**3. Shared Dependencies:**
- Dependencias compartidas entre host y remotes (Angular, RxJS, etc.)
- Evita duplicación de código y reduce bundle size
- Estrategia singleton para asegurar una sola instancia

### Comunicación Entre Microfrontends

#### 1. Event Bus Pattern (Observable Store)

```typescript
// libs/shared-data-access/src/lib/services/event-bus.service.ts
import { Injectable } from '@angular/core';
import { Subject, Observable } from 'rxjs';
import { filter, map } from 'rxjs/operators';

export interface BusEvent<T = any> {
  type: string;
  payload: T;
  timestamp: number;
}

@Injectable({
  providedIn: 'root',
})
export class EventBusService {
  private eventSubject = new Subject<BusEvent>();

  /**
   * Emite un evento al bus
   */
  emit<T>(type: string, payload: T): void {
    this.eventSubject.next({
      type,
      payload,
      timestamp: Date.now(),
    });
  }

  /**
   * Escucha eventos de un tipo específico
   */
  on<T>(eventType: string): Observable<T> {
    return this.eventSubject.pipe(
      filter((event) => event.type === eventType),
      map((event) => event.payload as T)
    );
  }
}

// Uso en microfrontend emisor (mfe-catalogos)
@Component({...})
export class CatalogoFormComponent {
  private eventBus = inject(EventBusService);

  onSave(catalogo: Catalogo): void {
    this.catalogosService.crear(catalogo).subscribe(() => {
      // Emitir evento para otros microfrontends
      this.eventBus.emit('catalogo:created', catalogo);
    });
  }
}

// Uso en microfrontend receptor (mfe-dashboard)
@Component({...})
export class DashboardComponent implements OnInit {
  private eventBus = inject(EventBusService);

  ngOnInit(): void {
    // Escuchar eventos de catálogos
    this.eventBus.on<Catalogo>('catalogo:created').subscribe((catalogo) => {
      console.log('Nuevo catálogo creado:', catalogo);
      this.refreshStats();
    });
  }
}
```

```typescript
// libs/shared-data-access/src/lib/store/auth.store.ts
import { Injectable, signal } from '@angular/core';

export interface AuthState {
  user: User | null;
  token: string | null;
  isAuthenticated: boolean;
}

@Injectable({
  providedIn: 'root',
})
export class AuthStore {
  // State compartido entre todos los microfrontends
  private state = signal<AuthState>({
    user: null,
    token: null,
    isAuthenticated: false,
  });

  // Getters
  user = computed(() => this.state().user);
  isAuthenticated = computed(() => this.state().isAuthenticated);

  // Actions
  setUser(user: User, token: string): void {
    this.state.set({
      user,
      token,
      isAuthenticated: true,
    });
  }

  logout(): void {
    this.state.set({
      user: null,
      token: null,
      isAuthenticated: false,
    });
  }
}

### 3. Servicio con RxJS para API

```typescript
// src/app/features/catalogos/services/catalogos.service.ts
import { Injectable, inject } from '@angular/core';
import { HttpClient, HttpParams } from '@angular/common/http';
import { Observable, BehaviorSubject, throwError } from 'rxjs';
import { map, catchError, tap, shareReplay } from 'rxjs/operators';
import { Catalogo, CatalogoRequest } from '../models/catalogo.model';
import { environment } from '@environments/environment';

@Injectable({
  providedIn: 'root'
})
export class CatalogosService {
  private http = inject(HttpClient);
  private apiUrl = `${environment.apiUrl}/catalogos`;
  
  // Estado reactivo con BehaviorSubject
  private catalogosSubject = new BehaviorSubject<Catalogo[]>([]);
  public catalogos$ = this.catalogosSubject.asObservable();
  
  // Cache con shareReplay
  private catalogosCache$?: Observable<Catalogo[]>;
  
  /**
   * Obtiene el árbol de catálogos
   */
  obtenerArbol(tipo?: string, soloActivos: boolean = true): Observable<Catalogo[]> {
    if (!this.catalogosCache$) {
      let params = new HttpParams();
      if (tipo) params = params.set('tipo', tipo);
      if (soloActivos) params = params.set('activo', 'true');
      
      this.catalogosCache$ = this.http.get<Catalogo[]>(`${this.apiUrl}/arbol`, { params })
        .pipe(
          tap(catalogos => this.catalogosSubject.next(catalogos)),
          shareReplay(1),
          catchError(this.handleError)
        );
    }
    
    return this.catalogosCache$;
  }
  
  /**
   * Crea un nuevo catálogo
   */
  crear(request: CatalogoRequest): Observable<Catalogo> {
    return this.http.post<Catalogo>(this.apiUrl, request)
      .pipe(
        tap(() => this.invalidateCache()),
        catchError(this.handleError)
      );
  }
  
  /**
   * Actualiza un catálogo existente
   */
  actualizar(id: string, request: CatalogoRequest): Observable<Catalogo> {
    return this.http.put<Catalogo>(`${this.apiUrl}/${id}`, request)
      .pipe(
        tap(() => this.invalidateCache()),
        catchError(this.handleError)
      );
  }
  
  /**
   * Elimina un catálogo
   */
  eliminar(id: string): Observable<void> {
    return this.http.delete<void>(`${this.apiUrl}/${id}`)
      .pipe(
        tap(() => this.invalidateCache()),
        catchError(this.handleError)
      );
  }
  
  /**
   * Invalida el cache
   */
  private invalidateCache(): void {
    this.catalogosCache$ = undefined;
  }
  
  /**
   * Manejo centralizado de errores
   */
  private handleError(error: any): Observable<never> {
    console.error('Error en CatalogosService:', error);
    const mensaje = error.error?.message || 'Error al comunicarse con el servidor';
    return throwError(() => new Error(mensaje));
  }
}

import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query'
import { toast } from '@/shared/hooks/useToast'
import { catalogosService } from '../services/catalogosService'
import type { Catalogo, CrearCatalogoRequest, ActualizarCatalogoRequest } from '../types/catalogo.types'

export function useCatalogos(tipo?: string) {
  const queryClient = useQueryClient()

  // Query: Obtener árbol de catálogos
  const { data: catalogos, isLoading, error } = useQuery({
    queryKey: ['catalogos', 'arbol', tipo],

    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['catalogos'] })
      toast({
        title: "Catálogo creado",
        description: "El catálogo se ha creado exitosamente.",
        variant: "success",
      })
    },
    onError: (error) => {
      toast({
        title: "Error al crear catálogo",
        description: error.message,
        variant: "destructive",
      })
    },
  })

  // Mutation: Actualizar catálogo
  const actualizarMutation = useMutation({
    mutationFn: (data: ActualizarCatalogoRequest) => catalogosService.actualizar(data),

      toast({
        title: "Error al actualizar catálogo",
        description: error.message,
        variant: "destructive",
      })
    },

import axios, { AxiosError, AxiosInstance, InternalAxiosRequestConfig, AxiosResponse } from 'axios'
import { getAuthToken, removeAuthToken } from '@/features/auth/lib/authStorage'

// Crear instancia de Axios
const apiClient: AxiosInstance = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8080/api/v1',
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
  },
})

// Interceptor de Request: Agregar token de autenticación
apiClient.interceptors.request.use(
  (config: InternalAxiosRequestConfig) => {
    const token = getAuthToken()
    if (token && config.headers) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  (error: AxiosError) => {
    return Promise.reject(error)
  }
)

// Interceptor de Response: Manejo de errores global
apiClient.interceptors.response.use(
  (response: AxiosResponse) => {
    // Extraer data de ApiResponse<T>
    if (response.data && 'success' in response.data) {
      return {
        ...response,
        data: response.data.data, // Retornar solo el data interno
      }
    }
    return response
  },
  async (error: AxiosError) => {
    // Manejar errores 401 (Unauthorized)
    if (error.response?.status === 401) {
      removeAuthToken()
      window.location.href = '/login'
    }

    // Manejar errores 403 (Forbidden)
    if (error.response?.status === 403) {
      // Redirigir a página de acceso denegado
      window.location.href = '/403'
    }

    // Transformar error a formato consistente
    const errorMessage = error.response?.data?.message || error.message || 'Error desconocido'
    
    return Promise.reject(new Error(errorMessage))
  }
)

export { apiClient }
